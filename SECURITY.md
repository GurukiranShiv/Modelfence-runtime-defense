# Security Policy

## Supported versions

Security fixes should target the latest version of this repository.

## Reporting vulnerabilities

Please report suspected vulnerabilities privately to the project maintainer instead of opening a public issue. Include:

- Affected component
- Steps to reproduce
- Expected impact
- Any proof-of-concept input/output
- Suggested remediation, if known

## Responsible use

This project is intended for defensive testing of systems you own or are authorized to assess. Do not use it to attack third-party systems without permission.

## Security scope

In scope:

- Unsafe tool execution in examples
- Report leakage of sensitive information
- CLI or dashboard vulnerabilities
- Dependency/security workflow issues
- Probe logic that creates unsafe defaults

Out of scope:

- Attacks against third-party model providers
- Unauthorized testing of public systems
- Social engineering or phishing
